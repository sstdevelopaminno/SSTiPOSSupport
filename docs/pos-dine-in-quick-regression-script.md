## เตรียมก่อนทดสอบ (60 วินาที)

- เปิดกะ (`shift`) แล้ว
- มีโต๊ะอย่างน้อย 2 โต๊ะ (`A`, `B`)
- มีสินค้าให้กดอย่างน้อย 2 รายการ

## Case 1: กดเร็ว/สลับโต๊ะเร็ว (2-3 นาที)

วัตถุประสงค์: จับอาการหน่วง ค้าง สลับโต๊ะผิด

ขั้นตอน:
1. เข้าโหมด `นั่งโต๊ะ`
2. กดโต๊ะ `A` แล้วใส่สินค้า 1 รายการ
3. กด `เลือกโต๊ะ` แล้วกด `B` ทันที
4. กด `เลือกโต๊ะ` แล้วกลับ `A`
5. ทำรอบ `A -> B -> A` ให้ครบ 3 รอบแบบเร็ว

ผ่านเมื่อ:
- ไม่มีปุ่มค้าง/กดไม่ติด
- ไม่เกิด error popup แปลกๆ
- กลับโต๊ะ `A` แล้วรายการสินค้าไม่หาย

ไม่ผ่านเมื่อ:
- หน่วงยาวผิดปกติจนใช้งานสะดุด
- คลิกแล้วไม่ตอบสนอง
- สลับแล้วข้อมูลโต๊ะ/ตะกร้าผิดโต๊ะ

ผลทดสอบ: `ผ่าน / ไม่ผ่าน`  
หมายเหตุ: `____________________________`

## Case 2: รีโหลดแล้วตะกร้าโต๊ะต้องไม่หาย (2-3 นาที)

วัตถุประสงค์: จับอาการ draft หายหลังรีโหลด/รีเซ็ต

ขั้นตอน:
1. เข้าโต๊ะ `A` ในโหมด `นั่งโต๊ะ`
2. ใส่สินค้า 2 รายการ (ยังไม่ชำระ)
3. รีโหลดหน้า 1 ครั้ง (`Ctrl+R`)
4. กลับเข้าโหมด `นั่งโต๊ะ` แล้วเลือกโต๊ะ `A`

ผ่านเมื่อ:
- รายการในตะกร้าของโต๊ะ `A` ยังอยู่ครบ
- จำนวน/ยอดรวมตรงก่อนรีโหลด

ไม่ผ่านเมื่อ:
- ตะกร้าว่างทั้งที่ก่อนรีโหลดมีรายการ
- รายการหายบางส่วนหรือยอดเพี้ยน

ผลทดสอบ: `ผ่าน / ไม่ผ่าน`  
หมายเหตุ: `____________________________`

## Case 3: ชำระเสร็จแล้วเด้งกลับหน้าเลือกโต๊ะ (2-3 นาที)

วัตถุประสงค์: ยืนยัน flow ปิดบิลโต๊ะและกลับหน้ารายการโต๊ะอัตโนมัติ

ขั้นตอน:
1. เข้าโต๊ะ `A` และใส่สินค้าอย่างน้อย 1 รายการ
2. กด `ชำระเงิน`
3. เลือกวิธีชำระ (เงินสดหรือโอน) และยืนยันจนสำเร็จ
4. สังเกตหน้าจอหลังชำระสำเร็จ

ผ่านเมื่อ:
- ระบบพากลับหน้า `รายการเลือกโต๊ะ` อัตโนมัติ
- โต๊ะที่ชำระแล้วไม่ค้าง state เก่าในตะกร้าฝั่งขวา

ไม่ผ่านเมื่อ:
- ชำระเสร็จแล้วยังค้างในหน้าเดิมโดยไม่กลับหน้าโต๊ะ
- ยังเห็นตะกร้าบิลเดิมค้างผิดเงื่อนไข

ผลทดสอบ: `ผ่าน / ไม่ผ่าน`
