## ก่อนเริ่ม

- [ ] เปิดกะ (`shift`) แล้ว
- [ ] มีโต๊ะอย่างน้อย 3 โต๊ะ (ว่าง 2, มีบิล 1)
- [ ] มีเมนูให้กดอย่างน้อย 2 รายการ

## A) เสถียรภาพการเลือกโต๊ะ (เสี่ยงสูง)

- [ ] A1 เปิดโหมด `นั่งโต๊ะ` แล้วกดโต๊ะว่าง -> เข้าหน้าโต๊ะได้ทันที ไม่ค้าง
- [ ] A2 กด `เลือกโต๊ะ` กลับหน้ารายการโต๊ะ แล้วกดโต๊ะเดิม -> เข้าได้ไว ไม่เด้ง error
- [ ] A3 กดสลับโต๊ะ A -> B -> A เร็วๆ 3 รอบ -> UI ไม่หน่วง/ไม่ค้าง/ไม่สลับผิดโต๊ะ

## B) ตะกร้าไม่หายเมื่อกลับโต๊ะเดิม (เสี่ยงสูงสุด)

- [ ] B1 โต๊ะ A ใส่สินค้า 2 รายการ (ยังไม่ชำระ) -> กลับหน้าโต๊ะ -> เข้าโต๊ะ A ใหม่
- [ ] B2 ผลต้องได้: สินค้าในตะกร้าโต๊ะ A ยังอยู่ครบ จำนวน/ยอดถูกต้อง
- [ ] B3 เข้าโต๊ะ B แล้วกลับมา A อีกรอบ -> ตะกร้าโต๊ะ A ยังเหมือนเดิม

## C) แยกโหมดไม่ปนกัน (นั่งโต๊ะ vs กลับบ้าน/เดลิเวอรี)

- [ ] C1 ขณะโต๊ะ A มีสินค้าในตะกร้า สลับไป `กลับบ้าน` -> ตะกร้าต้องไม่ติดมาจากโต๊ะ
- [ ] C2 สลับกลับ `นั่งโต๊ะ` แล้วเข้าโต๊ะ A -> ตะกร้าเดิมของโต๊ะ A ต้องกลับมาถูกต้อง
- [ ] C3 สลับไป `เดลิเวอรี` -> ห้ามเห็นตะกร้าบิลโต๊ะค้างข้ามโหมด

## D) ย้ายโต๊ะ (จุดเสี่ยง timeout/ข้อมูลหาย)

- [ ] D1 เปิดโต๊ะ A + ใส่สินค้า -> กด `ย้ายโต๊ะ` ไปโต๊ะ B -> ต้องสำเร็จ
- [ ] D2 หลังย้าย เข้าโต๊ะ B -> รายการ/ยอดต้องย้ายตามมาครบ
- [ ] D3 หลังย้าย ตรวจโต๊ะ A -> ต้องไม่เหลือบิล active เดิม
- [ ] D4 หากมีข้อความ timeout ให้กดใหม่ 1 ครั้ง -> ต้องไม่สร้างบิลซ้ำ/ข้อมูลเพี้ยน

## E) ปุ่มสำคัญต้องกดได้จริง

- [ ] E1 ปุ่ม `เลือกโต๊ะ` กดได้ทุกครั้ง ไม่หน่วง
- [ ] E2 ปุ่ม `พักบิล` กดได้ และดึงกลับได้
- [ ] E3 ปุ่ม `ยกเลิกบิล` (ก่อน/หลังสร้างออเดอร์ตามสิทธิ์) ทำงานถูกต้อง
- [ ] E4 ปุ่ม `ชำระเงิน` ไม่ค้าง/ไม่เด้งเงื่อนไขผิดโต๊ะ
